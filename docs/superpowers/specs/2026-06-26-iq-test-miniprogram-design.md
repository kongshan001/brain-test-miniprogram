# 脑力测试微信小程序 — 设计规格

> 2026-06-26 | MVP 方案一：单题库极简流

## 一、产品概要

一款微信小程序 IQ 测试应用，用户进入后选择难度、连续回答 10 道趣味烧脑题，获得分数和等级，生成分享海报进行社交传播。

- **目标用户**：大众娱乐 + 学生/家长成长评估
- **变现策略**：MVP 阶段纯免费，专注获客体验
- **社交范围**：MVP 轻分享（结果海报），预留排行榜/PK/社区

## 二、技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 前端框架 | Taro 3 + React + TypeScript | 跨端编译，后续可迁移到其他平台小程序 |
| 后端 | 微信云开发（CloudBase） | 云数据库 + 云函数 + 云存储，免运维 |
| 海报生成 | 云函数 + Canvas | 服务端渲染分享海报 |

## 三、设计系统

### 3.1 风格
- **Style**: Vibrant & Block-based — 大胆、活力、几何色块
- **配色**:

| 角色 | 色值 | 用途 |
|------|------|------|
| Primary | `#2563EB` | 主色调、选中态、链接 |
| Secondary | `#7C3AED` | 渐变辅助、结果页背景 |
| Accent/CTA | `#F59E0B` | 主按钮、倒计时 |
| Background | `#EFF6FF` | 页面底色 |
| Foreground | `#0F172A` | 正文 |
| Success | `#22C55E` | 答对反馈 |
| Error | `#EF4444` | 答错反馈 |

- **字体**: Fredoka（标题）+ Nunito（正文）— 活泼友好
- **间距**: 8dp 系统（8 / 16 / 24 / 32 / 48）
- **动画**: 150–300ms 过渡，ease-out 进入 / ease-in 退出

### 3.2 核心交互原则
- 追碰目标 ≥ 44×44pt
- 选项选中 0.3s 平滑反馈（绿正确/红错误）
- 骨架屏加载（>300ms 操作）
- 每屏一个主 CTA（金色渐变按钮）

## 四、页面架构

```
首页 (选难度) → 答题页 (10题计时) → 结果页 (得分+等级) → 分享海报 (裂变)
```

### 4.1 首页
- 蓝紫渐变 Hero 区：品牌名 + slogan + 难度选择
- 三个难度卡片横向排列：简单 / 中等（默认推荐，高亮）/ 困难
- 金色渐变 CTA 按钮："开始测试"
- 底部轻提示："10 道题 · 约 3 分钟"

### 4.2 答题页
- 顶栏：题号标签（金色圆角方块）+ 渐变色进度条 + 倒计时
- 题干区：浅蓝灰底色卡片，大图标 + 文字题干
- 选项：4 个圆角按钮，点击后 0.3s 过渡反馈（绿色正确/红色错误），自动跳下一题
- 每题限时 30s，超时自动跳过

### 4.3 结果页
- 上半部：紫蓝渐变背景，大数字分数（80px）+ 毛玻璃等级标签
- 下半部：白色统计卡片（答对/答错/均速 三列分割线布局）
- 主按钮："生成分享海报"（金渐层）
- 次按钮："再做一次"（蓝色文字无背景）

### 4.4 分享海报
- 卡片式布局：渐变品牌区（分数 + 等级 + 统计） + 小程序码 + 行动号召
- 云函数 Canvas 渲染，保存相册或转发好友

## 五、数据模型

### questions 表
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 题目 ID |
| type | string | 题型预留，MVP 默认 `logic` |
| difficulty | number | 难度 1-3 |
| content | object | 题干文本 + 可选图片 URL |
| options | array | 4 选项 `{ key, text, image? }` |
| answer | string | 正确选项 key |
| time_limit | number | 限时秒数，默认 30 |

### results 表
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 记录 ID |
| openid | string | 用户标识 |
| score | number | 得分 0-100 |
| difficulty | number | 所选难度 |
| answers | array | 每题作答 `{ question_id, chosen, correct, time_spent }` |
| created_at | date | 完成时间 |

### share_posters 表
| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 海报 ID |
| result_id | string | 关联成绩 |
| cloud_path | string | 云存储路径 |
| created_at | date | 生成时间 |

## 六、评分算法

| 维度 | 计算方式 |
|------|----------|
| 正确得分 | 答对 +10 分，答错 0 分（满分 100） |
| 速度奖励 | 答对时剩余时间 > 20s → +0.5；> 10s → +0.25 |
| 最终得分 | 四舍五入取整 |

### 等级对照

| 分数 | 等级 | 文案 |
|------|------|------|
| 90-100 | S | 脑力天花板 |
| 70-89 | A | 才思敏捷 |
| 50-69 | B | 潜力无限 |
| 30-49 | C | 继续修炼 |
| 0-29 | D | 今日状态不佳 |

## 七、题库策略

- MVP: 150 题（每难度 50 题），每次随机抽 10 题
- 题目来源：手动录入经典烧脑题型（数列、逻辑推理、图形识别）
- 扩展预留：`type` 字段支持后续分类（逻辑/数字/空间/记忆）

## 八、扩展预留

- 题型维度分类（logic/number/spatial/memory）→ 分项得分 + 雷达图
- 好友排行榜 + PK 挑战模式
- 每日打卡 + 智力日历热力图
- 多端迁移（Taro 编译到支付宝/抖音小程序）

## 九、约束与限制

- 准确率仅作娱乐参考，不适用于临床诊断、人才选拔等正式场景
- 微信云开发免费额度：数据库 2GB、云函数 4 万次/月、存储 5GB
- 小程序需备案域名 + HTTPS
